软件工程师个人品质、团队精神，你都做到了哪些？

2018-10-03
  • job
阅读量：次 | 字数：778 | 阅读时长：大约2分钟

如题

团队成长与个人成长

  1. 单兵作战能力，独当一面，战无不胜~~~
  2. 如何成长快一点？善于总结，乐于分享。工作追求，张弛有度，轻松愉快，同时要保证有一个正常合理的产出，结果导向。

code

  1. DRY（Don’t repeat yourself）
  2. 约定胜于配置
  3. 让每个类尽量简单，尽量只做一件事
  4. sql 查询，少用 inner join
  5. 变量/类/文件，命名，见名知义
  6. 变量/文件，驼峰命名规则，类命名要首字母大写
  7. tab -> 四个空格，保持 idea / webStorm 默认配置
  8. 文件编码：utf8

遇到技术难题，卡住时

  1. 及时沟通，好调整工作计划。
  2. 安排解决问题的时间。
  3. 注意看提示，仔细看，根据提示操作，尝试，95% 以上的问题都能找到解决问题的办法。
  4. 提问的智慧

软件工程师品质

  1. 关注细节
  2. 严谨
  3. 责任心
  4. 细心
  5. 耐心
  6. 意识
    • 产品意识
    • 维护意识
    • 换位思考
  7. 进度与质量
  8. 应该偷懒，正确的偷懒
  9. 理解
    • 业务
    • 代码
    • 时间关系
  10. 多花心思
  11. 勤奋
  12. 能吃苦
  13. 肯专研
  14. 能解决实际问题
  15. 学习能力强

个人品质修为

  1. 勿以恶小而为之，勿以善小而不为。惟贤惟德，能服于人。

    系蜀汉先主刘备所言。刘备去世前给其子刘禅的遗诏中的话。刘禅 –> 乐不思蜀

  2. 细节决定成败。
  3. 遇到问题，解决它，度娘，团队，不能将就。
  4. 多测试，多思考，高要求。
  5. 要知道自己的短板在哪儿，及时弥补。
  6. 培养快速解决问题的能力，厚积薄发。
  7. 善于发现问题。
  8. 由要我做什么，到我要做什么。

由“收到请回复”说开

  1. 别人让你做什么事，做好之后，给一个反馈，这个非常有必要。
  2. 收到消息，回复一个 【1】 或者 【ok】 或者【收到】，是对别人的尊重。
  3. 需要别人帮忙，有结果了，告之一声，也是对别人的尊重。
  4. 工作进展，及时沟通，不要等问了再说。

日常沟通

沟通要到位，理解了需求再动手，不清楚的及时问。

团队精神

  1. 仔细研读代码规范。
  2. 理解约定胜于配置的含义及好处。
  3. 代码的可读性、可扩展性，是代码质量的重要指标。
  4. 主动性，积极性，表现在哪些方面？
  5. 团队精神、团队协作能力，表现在哪些方面？
  6. 沟通能力，体现在哪些地方？

团队氛围

积极的，向上的，正能量的，活泼的，紧张的，放松的

绩效目的

  1. 公平、公正，付出（产出）与回报成正比。
  2. 让肯努力、真正有能力的人得到该有的回报，激励大家共同进步，淘汰影响团队发展的人。
  3. 日积月累，让别人永远也到达不了你的高度。

Nothing worth having comes easy.

值得拥有的东西，永远都来之不易。
你今天的日积月累，早晚会成为别人的望尘莫及。
