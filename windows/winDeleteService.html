问题原因及解决办法

  1. 导致 windows service 不能部署，也不能被删除，使用 sc delete serviceName 命令也不奏效。是因为 windows service database 缓存的原因，简单粗暴的办法就是重启操作系统。
  2. 我遇到这个问题的原因是我的 windows service 控制面板没有完全关闭，之前打开的 nginx 属性窗口还开着，又点开了另外一个服务窗口，导致 nginx 服务信息一下存在于 windows 缓存当中，关闭 windows service 控制面板，问题解决。

  3. 通常，可以尝试下面的办法解决这种问题：

    1. 查找 windows service 的 PID：运行 queryex service_name，例如：

    2. 如果 PID 项不为0，则杀掉进程：taskkill /PID service_pid /f

    3. 如果 PID 为0，则很有可能遇到了跟我一样的问题，需要关闭所有 windows service 控制面板。

如何在 windows 下安装和删除 nginx 服务，请看我的另外一篇文章：把 Nginx 安装为 Windows 的一个服务
